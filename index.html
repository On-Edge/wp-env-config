wp-env-config

An addon that makes it easy to run the same WordPress site across multiple environments.

Who needs this?

If you're a WordPress developer, you probably have multiple copies of the same site running across a number of different server configurations: your local installation, a staging server, the live site, etc...

Managing these settings in 1 wp-config.php file can be kind of a problem when you want to keep your code in version control. If you're not doing that already, stop reading this and figure that out first. This is a good place to start.

Already using git/svn/... to manage your code? Good. This will help you keep your WordPress config a bit cleaner and easier to manage.

How does it work?

First, you edit your wp-config.php file so that it will include config.master.php, which in turn will look for conf.env.php. In that file, you'll define the enviromnent you have/need, based on the URL from which the website is served.

if (!defined('ENV')) {
    switch (strtolower($_SERVER['HTTP_HOST'])) {
        case 'website.tld' :
        case 'www.website.tld' :
            define('ENV', 'prod');
            define('ENV_FULL', 'Production');
            define('ENV_DEBUG', false);

In the example above, config.prod.php will be used when the site is visited through website.tld or www.website.tld.

If no URL matches, the settings in config.local.php will be used. The repository contains a more extended readme that details how to install and adapt this code to make it work with your WordPress install.
